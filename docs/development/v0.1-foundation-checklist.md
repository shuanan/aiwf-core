# AIWF Core v0.1 Foundation Checklist

This checklist defines the minimum foundation for AIWF Core before moving to real downstream pilots.

## Completed foundation

- Standalone `aiwf-core` repository exists.
- Initial scaffold is committed.
- Local validation script exists.
- Validation documentation exists.
- GitHub Actions runs validation on push and pull request.
- YAML parsing runs in CI with PyYAML.
- Internal registry/release references are validated.
- Quickstart documentation exists.
- Minimal downstream adapter example exists.

## Required checks

Before declaring the v0.1 foundation ready:

```bash
bash scripts/validate-aiwf-core.sh
git status --short
```

Expected validation:

```text
validate-aiwf-core: 26 passed, 0 failed, 0 skipped
```

## Not part of v0.1 foundation

- Slack automation.
- Linear automation.
- Codex execution.
- Claude `-p` execution.
- Claude Code hook installation.
- `.claude/settings.json`.
- Auto-merge.
- Auto-fix.
- Production deployment.
- Real downstream adoption.

## Next phase

After v0.1 foundation is stable, the next safe phase is to improve validation and examples before any automation:

1. Validate example adapter structure.
2. Add schema examples.
3. Add release checklist.
4. Prepare one notify/manual downstream pilot.
